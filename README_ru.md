# SSH Proxy Tray

Простое приложение в трее для включения локального SOCKS‑прокси через SSH. Нажмите на значок трея, чтобы включить/выключить прокси или переключить режим «системный прокси» / «только локально».

## Обзор
- Запускает локальный SOCKS‑прокси через SSH и показывает его статус в трее
- Опция «Use as system proxy» — автоматически настраивает системный прокси GNOME
- Параметры автозапуска и автоподключения

## Перед началом (SSH)
Задайте сервер один раз в `~/.ssh/config` и обращайтесь к нему по алиасу (по умолчанию приложение использует алиас `nl`, порт SOCKS — `1080`). Пример:

```
Host nl
  HostName 203.0.113.23
  User alice
  Port 22
  IdentityFile ~/.ssh/id_ed25519
  ServerAliveInterval 60
  ServerAliveCountMax 3
  Compression yes
```

- Замените `203.0.113.23` на реальный IP или имя вашего сервера.
- Убедитесь, что ключ существует и им можно пользоваться (рекомендуется ssh‑agent).
- Если используете другой алиас, укажите его в Настройках.
- Если порт 1080 занят, выберите другой порт в Настройках.

## Установка и запуск
- Чтобы собрать Debian‑пакет, выполните `./build.sh`, затем установите получившийся `.deb` из каталога `build-deb/...`.
- После установки запустите «SSH Proxy Tray» из меню приложений или выполните `ssh-proxy-tray` в терминале.

## Использование
- SSH Proxy On/Off: запустить или остановить прокси
- Use as system proxy: переключение между системным и локальным режимами
- Autostart / Auto‑connect: контроль поведения при старте
- Settings…: изменить SSH‑алиас и SOCKS‑порт

## Диагностика
- «Порт уже используется»: выберите другой SOCKS‑порт в Настройках
- «Failed to connect»: проверьте работу `ssh <alias>` и содержимое `~/.ssh/config`
- Системный прокси не изменился: убедитесь, что доступен GNOME/`gsettings`
